IMPROVEMENT — 큰 그림

먼저, 이게 어떻게 실시간으로 가능한가

화면은 픽셀 약 200만 개. 지형 메시는 정점(vertex) 약 14.8만 개. 이걸 초당 60번 다시 계산하는 건 CPU로는 불가능해요.

GPU(그래픽 칩)는 이 수백만 개를 동시에 병렬로 처리하도록 만들어진 하드웨어입니다. 각 점/픽셀마다 작은 프로그램(셰이더)이 한 번씩 돌아요.

비유 — 한 명이 200만 장을 칠하면 하루, 200만 명이 한 장씩 동시에 칠하면 1초. GPU가 그 "200만 명"입니다.

그래서 이 발표의 거의 모든 마법은 셰이더(GLSL) 안에서 벌어집니다.

Fragment shader — 픽셀 1개당 실행
void main() {            // 200만번 동시에
  gl_FragColor = 색;   // "나는 이 색"
}
Vertex shader — 정점 1개당 실행
void main() {              // 14.8만번 동시에
  pos.z += 높이;          // "나는 여기로"
  gl_Position = 투영(pos);
}

IMPROVEMENT — 기반 개념

텍스처는 "그림"이 아니라 "숫자 격자"

GPU에게 텍스처는 픽셀마다 숫자 4개(RGBA)가 든 2D 표일 뿐. 색으로 안 읽고 다른 의미로 써도 됩니다.

  • 높이맵: 밝기 = 높이. 흰색=높음, 검정=평지. → 산의 능선
  • 흑백 표면맵: 명암 디테일
  • 속도장(유체): R,G = 흐름 방향 (색 아님)
Terrain.js — 높이맵 읽기
float h = texture2D(높이맵, uv).r;  // 밝기 0~1
pos.z += h * 산높이;             // 그만큼 위로
우리 높이맵 4종: landscape-disp · vertex-map · terrain005 · landscape4k (원본이 실제로 쓰는 것들)
0
슬라이더를 올리면 위 흑백 띠(높이맵)의 밝기대로 아래가 산으로 솟아오릅니다. = 정점 변위(vertex displacement).
LAYER 1 · 지형

평면을 산으로 — 변위 + 두 산 블렌드

세분화된 평면(800×1200, 정점 14.8만)의 각 정점을 높이맵 밝기만큼 밀어올립니다. 그리고 산을 모프하려고 항상 두 지형(A·B)을 섞어 둡니다.

Terrain.js (vertex)
// 두 개의 완성된 지형을 uMorph로 섞음
float worldH(uv) {
  return mix(
    bh(uv, alphaA, offA) * 높이A,   // 출발 산
    bh(uv, alphaB, offB) * 높이B,   // 도착 산
    uMorph);                        // 0→1
}
pos.z += worldH(uv);
핵심: offset을 연속으로 밀면 텍스처가 "미끄러져" 보임 → 대신 두 상태를 cross-fade 하면 봉우리가 제자리에서 솟고 꺼지며 모프 (LAYER 8에서 자세히).
// 변위만 (색·조명 OFF) — 형태만 보면 진짜 3D 산
LAYER 3 · 조명

빛과 그림자로 입체감 — 법선 조명

높이만으론 납작해요. 각 점이 어느 방향을 향하는지(법선)를 이웃 높이차로 구하고, 햇빛 방향과 비교해 밝기를 정합니다.

Terrain.js (fragment)
float diff = dot(법선, 햇빛방향);
// 정면=1(밝음), 직각=0(그림자)
col = albedo * (0.25 + 0.8*diff);

법선은 버텍스에서 높이 기울기로 계산:

Terrain.js (vertex)
hX = worldH(uv + 오른쪽);
hY = worldH(uv + 위쪽);
vNormal = normalize(vec3(
  -(hX-h)/dx, 1.0, -(hY-h)/dy));
55°
햇빛 방향을 돌리면 같은 언덕의 밝은 면/그늘이 바뀝니다. 이 명암이 우리 눈에 "3D"로 읽혀요.
LAYER 13 · 숲

표면을 숲으로 — Voronoi 나무 도장

나무를 물체로 심지 않고(InstancedMesh ✕), 표면 프래그먼트에서 절차적으로 그립니다. UV를 격자로 나눠 각 칸에 점을 찍는 Voronoi로 둥근 나무 도장을 만들어요.

Terrain.js — voronoi 나무
vec2 fuv = vUv * 240.0;   // 칸 밀도
// 가장 가까운 셀까지 거리 md
for (y=-1..1) for (x=-1..1) {
  점 = hash(셀+이웃);
  md = min(md, 거리²);
}
float dab = smoothstep(sz,*, sqrt(md)); // 둥근 나무
  • 나무마다 랜덤 크기·초록·밝기 + 드물게 벚꽃 핑크·노랑
  • 색은 높이로 그라데이션(계곡 진초록 → 능선 연두)
20
Voronoi 셀마다 둥근 나무 도장 하나. 밀도를 올리면 작고 촘촘한 숲이 됩니다. (실제 셰이더와 같은 원리)
LAYER 13 · 숲

숲이 "평면 만화"가 되지 않으려면

나무 도장이 제각각 고대비면, 그 노이즈가 산의 큰 명암(3D 형태)을 덮어 평평하게 보여요. 그래서:

  • 나무는 은은한 질감만 (틈도 새까맣지 않게, 부드러운 가장자리)
  • 큰 색은 높이(형태) 가 주도 → 그 위에 강한 조명
  • 결과: 숲을 켜도 산의 입체감 유지
Terrain.js
// 형태가 주도하는 초록 베이스

forest = baseGreen * (0.74+dab*..) * 랜덤밝기;
// 그 위에 강한 빛/그림자
lit = forest * (0.25 + 0.8*diff);

LAYER 12 · 암석 · 폭포

경사로 암석, 골짜기로 폭포 — 자동 배치

암석 = 가파른 곳

경사 = 1 − 법선.y
float slope = 1.0 - n.y;          // 0평지 1수직
float steep = smoothstep(.4,.54,slope);
색 = mix(숲, 바위색, steep);   // 절벽엔 바위
나무 *= 1.0 - steep;            // 나무는 피해서

폭포 = 오목한 골짜기

곡률(Laplacian)로 골 찾기
// 이웃합-4*중심 >0 = 오목(골)
vConcavity = hX+hXn+hY+hYn - 4.0*h;
float valley = smoothstep(.15,1., vConcavity);
폭포 = valley * steep * 세로줄기 * 흐름;
  • 나무가 암석을 피함 — 같은 경사 마스크(steepM)로 절벽엔 나무 제거 → 바위가 깔끔히 드러남
  • 폭포는 오목한 골에만 — 곡률(Laplacian)이 양수인 채널 + 가파름 + 세로 줄기
  • 중력 방향으로 흐름 — 높이(vHeight) 따라 아래로 애니메이션, 파란 물색
  • 조명 뒤에 덧칠 → 절벽 그늘에서도 폭포가 밝게 보임
전부 지형의 기하(경사·곡률)에서 자동 유도 — 위치를 손으로 안 찍었어요.

LAYER 8 · 클릭 모프

메뉴 클릭 → 다른 산으로 (미끄러짐 없이)

메뉴를 클릭하면 그 프로젝트 산으로 전환, 다음 클릭 전까지 유지. 핵심은 "두 상태 cross-fade"로 미끄러짐을 없앤 것.

main.js — applyPreset
// 현재 B를 A로 스냅샷, 새 프리셋을 B로
A = B;  B = 새프리셋;

두 개의 완성된 지형을 높이만 보간 → 봉우리가 제자리에서 솟고 꺼짐 (UV를 밀면 텍스처가 옆으로 흐르는 "슬라이딩"이 안 생김).

발견: 표면색의 정체

원본 산 표면은 알록달록 텍스처가 아니라 흑백 마블 + fog색이었어요. 컬러 텍스처를 채도 0으로 깔고, 색은 fog에서. (그래서 안 알록달록)

IMPROVEMENT — 셰이더 심화

Terrain 프래그먼트 — 한 픽셀이 색을 정하는 전 과정

void main() { // 이 픽셀의 최종 색을 계산 1. t = 높이를 0~1로 (계곡→능선) 2. marble = 흑백텍스처 × 무드색 // 미스티 모드(숲 OFF) 3. forest = Voronoi 나무 도장 + 높이별 초록 + 벚꽃 // 숲 모드 └ slope(경사)로 절벽엔 나무 제거 4. rock = 가파른 곳 회청색 바위 (steepM) 5. waterfall = 오목한 골(곡률) + 세로줄기 → wf 마스크 보관 6. albedo = mix(marble, forest, uForestOn) // 13번 토글 7. lit = albedo × (0.25 + 0.8·diff) // 강한 빛/그림자 8. col = lit + 파란 폭포(wf) // 조명 뒤 덧칠 9. col = mix(col, 안개색, fog) // 대기 원근 gl_FragColor = vec4(col, 1.0); }

토글들(uForestOn·uRockOn·uGradeOn·uLightOn·uFogOn)은 전부 0/1 uniform으로 mix를 켜고 끕니다 → 패널의 13겹이 이 한 셰이더 안에서 합쳐짐.

IMPROVEMENT — 셰이더 심화

색은 "높이"가 만든다

숲의 초록도, 미스티 산의 색도 전부 높이(vHeight)로 두 색을 섞어 만듭니다. 계곡은 진하게, 능선은 밝게.

Terrain.js
float t = smoothstep(0., .55, vHeight);
vec3 green = mix(진초록, 연두, t);

여기에 거리 안개까지 더하면, 가까운 골은 진한 초록 / 먼 능선은 옅은 안개초록 = 레퍼런스 채색화의 겹친 산 느낌.

LIMITATION — 한계 & 방향

지금의 한계 — 솔직하게

  • 숲 모양이 너무 일률적·이상함 — Voronoi 나무 패턴이 반복·균일해 부자연스럽고 평면적. 군집·크기·수종 다양성, 능선 실루엣 강조가 필요
  • 라이팅 — 단일 방향광뿐. 그림자·앰비언트 오클루전·시간대(노을/아침) 등 더 연구 필요
  • 시점 이동 — 거의 고정. 카메라 무빙/스크롤 연출·인터랙션이 부족
  • 패턴 — 나무·바위 배치가 절차적이라 균일. 손으로 그린 듯한 변주가 필요
  • 안개 — 단순 거리 페이드. 볼류메트릭 안개·God ray 연구
  • 물 · 폭포 — 폭포가 아직 표면 텍스처라 어색. 별도 지오메트리/파티클, 물 반사·굴절 고도화 필요
  • 추가 모티프 — 학·구름·동물 등 채색화 요소, 필름 그레인으로 질감
  • 성능 — 정점/유체 부하 최적화
핵심 방향: "절차적이라 균일하다"는 한계를 깨는 것 — 더 풍부한 변주(노이즈·군집·수작업 느낌) + 연출(조명·카메라·물·안개)을 연구해 양식화된 자연의 완성도를 높이기.
