— 12겹의 비밀

이 배경은 사진도 동영상도 아닙니다. 컴퓨터가 매 순간(초당 60번) 수백만 개의 점을 직접 계산해 그리는 실시간 3D예요. 그 과정을 12개의 얇은 층으로 쪼개서, 한 겹씩 어떻게 쌓이는지 보여드립니다.

큰 그림

먼저, 이게 어떻게 가능한가

화면은 점(픽셀) 200만 개로 이뤄져 있어요. 컴퓨터의 그래픽 칩(GPU)은 이 200만 개를 거의 동시에 계산하도록 만들어진 특수 부품입니다.

비유 — 한 명이 200만 장의 색칠 공책을 칠하면 하루가 걸리지만, 200만 명이 한 장씩 동시에 칠하면 1초도 안 걸려요. GPU가 바로 그 "200만 명"입니다.

각 픽셀마다 "너는 무슨 색이 될래?"를 정하는 작은 프로그램이 도는데, 이걸 셰이더(shader)라고 합니다. 이 발표의 거의 모든 마법은 셰이더 안에서 일어나요.

  • 레이어 1 — 진짜 3D 산 (입체 지형 + 카메라)
  • 레이어 2 — 그 위에 덧칠하는 효과 (안개·발광)
// 우리가 복제한 최종 결과 — 이걸 12겹으로 분해합니다
셰이더 = 픽셀마다 도는 함수
void main() {           // 픽셀 1개마다 실행, 200만번 동시에
  gl_FragColor = 색깔;  // "나는 이 색이야"
}
STEP 01 · 지형

평평한 판을 산으로 — 변위(Displacement)

핵심 발상: 텍스처(이미지)는 그림이 아니라 "숫자표"다. 흑백 이미지의 밝기를 "높이"로 해석합니다. 흰색=높음, 검정=낮음.

잘게 쪼갠 평평한 판의 각 점을, 그 위치의 이미지 밝기만큼 위로 밀어올리면 → 산맥이 됩니다.

비유 — 핀아트 장난감(손을 대면 핀이 솟는 것)과 같아요. 이미지가 "어디를 얼마나 밀지"를 알려주는 거푸집입니다.
0
슬라이더를 올리면 위 흑백 띠(높이맵)가 아래에서 산으로 솟아오릅니다. 밝은 곳일수록 높이 올라가죠.
// 색·조명 끄고 본 '날것의 지형' — 형태만
버텍스 셰이더 (점의 위치를 옮김)
float 높이 = texture(높이맵, 위치).r;  // 이미지 밝기 0~1
점.z += 높이 * 108.0;            // 그만큼 위로
STEP 02 · 색

높이에 따라 색칠하기 — 컬러맵

이제 산이 회색 덩어리니까 색을 입힙니다. 색은 사진에서 그대로 가져오지 않아요. 그러면 알록달록 부자연스럽거든요.

대신 높이에 따라 두 색 사이를 부드럽게 섞습니다 — 낮은 계곡은 어둡게, 높은 능선은 밝게.

비유 — 물감 두 개(계곡색·능선색)를 높이라는 비율로 섞는 것. 0%면 계곡색, 100%면 능선색, 중간이면 반반.
mist
슬라이더를 옮기면 같은 산이 회색→주황→보라로 "분위기"가 바뀝니다. 실제 사이트에서 글씨에 마우스를 올리면 일어나는 일이에요.
// 높이 기반 색 적용 — 아직 납작함(조명 전)
프래그먼트 셰이더 (픽셀 색 결정)
색 = mix(계곡색, 능선색, 높이);  // 높이로 섞기
STEP 03 · 조명

빛과 그림자로 입체감 — 법선 조명

지금까지는 높이로 색만 칠해서 납작했어요. 진짜 산처럼 보이게 하는 비밀이 이겁니다.

각 지점이 어느 방향을 향하는지(법선)를 계산하고, 햇빛 방향과 비교합니다. 햇빛을 마주 본 면은 밝게, 반대 면은 어둡게.

비유 — 손전등을 울퉁불퉁한 벽에 비추면, 빛을 향한 면은 환하고 그늘진 면은 어둡죠. 그 명암이 '입체'로 보이게 합니다.
  • 면이 햇빛을 정면으로 → 밝음
  • 면이 햇빛에 등 돌림 → 그림자
  • 이 명암 차이 = 우리 눈이 "3D"로 읽는 정보
// 조명 ON — 갑자기 진짜 바위산이 됨
빛의 양 = 면방향 · 햇빛방향
float 밝기 = dot(면방향, 햇빛방향);
// 두 방향이 일치할수록 1(밝음), 반대면 0(그림자)
색 = 색 * (0.5 + 0.5 * 밝기);
STEP 04 · 안개

멀수록 뿌옇게 — 대기 원근(안개)

실제로 멀리 있는 산은 공기층 때문에 흐릿하고 옅게 보입니다. 이걸 흉내 내요.

카메라에서 멀어질수록 그 픽셀의 색을 안개색 쪽으로 섞습니다. 가까운 산은 또렷, 먼 산은 하늘로 녹아듦.

비유 — 멀리 있는 산일수록 뿌연 유리를 한 겹씩 더 끼우는 것. 거리만큼 흐려집니다.

이 한 가지가 장면 전체의 "몽환적·공기감"을 책임집니다. 안개가 없으면 그냥 딱딱한 3D 모델처럼 보여요.

// 먼 능선이 헤이즈로 사라지며 깊이감 생김
거리에 비례해 안개색과 섞기
float 거리 = 카메라에서_이_점까지;
float 안개량 = smoothstep(560.0, 1450.0, 거리);
색 = mix(색, 안개색, 안개량);  // 멀수록 안개색
STEP 05 · 하늘

배경 하늘 — 그라데이션 돔

장면 전체를 감싸는 거대한 공(球) 안쪽에 하늘색 그라데이션을 칠합니다. 카메라는 그 공 안에 있죠.

위(천정)는 진한 하늘색, 지평선 근처는 밝은 헤이즈 — 그래서 산이 밝은 지평선을 배경으로 실루엣이 됩니다.

비유 — 천장이 하늘로 칠해진 돔 영화관 안에 들어와 있는 것. 어딜 봐도 하늘이 배경에 깔려요.

안개색과 지평선 하늘색을 같게 맞춰서, 산이 하늘로 자연스럽게 녹아들게 했습니다.

// 하늘 추가 — 산이 하늘을 배경으로 떠오름
바라보는 높이로 하늘색 결정
float 위쪽정도 = 보는방향.y;        // -1(아래)~+1(위)
하늘색 = mix(지평선색, 천정색, 위쪽정도);
STEP 06 · 물

계곡에 흐르는 물 — 반사 + 해안선

물 평면을 일정 높이에 깔되, 그 아래 산의 높이를 읽어서 산보다 낮은 곳에만 물이 보이게 합니다. 그래서 해안선이 산을 따라 구불구불해져요(딱딱한 직선이 아니라).

  • 해안선 — 물이 산 높이를 읽어 얕은 곳엔 거품
  • 반사 — 비스듬히 볼수록 하늘을 비춤(프레넬)
  • 잔물결 — 무늬를 어긋나게 두 번 흘려 끝없이 일렁
비유 — 욕조에 물을 받으면 높은 바위는 섬처럼 남고 낮은 곳만 잠기죠. 물이 지형 높이를 "읽어서" 그렇게 채웁니다.
// 물 추가 — 계곡에 물, 봉우리는 섬처럼 솟음
수심 = 물높이 − 그 아래 지형높이
float 수심 = 물높이 - 지형높이;
if (수심 < 0.0) discard;   // 산이 더 높음 → 물 안그림(땅)
거품 = (수심이 얕을수록 ↑);  // 해안선 거품
STEP 07·08 · 움직임 & 모핑

살아있게 + 산이 다른 산으로 변신

07 미세 움직임 — 시간에 따라 흐르는 노이즈를 아주 살짝 더해 표면이 천천히 숨 쉬듯 일렁이게.

08 모핑 — 산은 여러 높이맵을 미리 준비해두고, 글씨에 마우스를 올리면 두 높이맵 사이를 천천히 섞습니다. 그러면 봉우리들이 새 위치로 기어가며 다른 산이 됩니다.

0%
슬라이더를 끌면 산 A가 산 B로 부드럽게 변형됩니다. 봉우리가 갈아끼워지는 게 아니라 "이동"하는 게 핵심.
// 글씨 호버 → 산·색 동시에 다른 프로젝트로
두 높이맵을 비율로 섞기
높이 = mix(산A높이, 산B높이, t);
// t를 0→1로 천천히 올리면(GSAP) 봉우리가 이동
STEP 09 · 유체

마우스 따라 휘몰아치는 안개 — 유체 시뮬레이션

가장 정교한 부분. 진짜 물·연기의 물리 방정식을 GPU로 매 프레임 풉니다.

  • 속도장 — 화면 곳곳의 "흐르는 방향"을 저장한 표
  • 염료장 — 그 흐름에 풀린 "잉크(눈에 보이는 안개)"
  • 마우스 — 커서 위치에 힘과 잉크를 주입(splat)
비유 — 강물(속도장)은 안 보이지만, 잉크(염료장)를 떨어뜨리면 물살을 따라 퍼지며 흐름이 보이죠. 마우스가 그 잉크를 떨어뜨리는 손가락입니다.
위 칸에 마우스를 문질러 보세요. 커서를 따라 잉크가 번지고 흐릅니다 (실제 유체 알고리즘의 단순 버전).
// 산 위에 흐르는 안개 한 겹이 더 얹힘
이류(Advection): 잉크가 흐름을 따라감
// "이 잉크는 방금 전에 어디서 흘러왔지?"를 거꾸로 추적
vec2 출발점 = 현재위치 - 속도 * 시간;
새_잉크색 = texture(염료장, 출발점);
STEP 10·11·12 · 마무리 후처리

합성 · 발광 · 톤보정

마지막으로 완성한 그림 전체에 사진 보정 같은 마무리를 입힙니다.

  • 10 합성 — 유체 안개를 산 위에 덧칠 + 흐름으로 화면 살짝 왜곡
  • 11 블룸(발광) — 밝은 부분을 번지게 해 몽환적 빛
  • 12 톤보정 — 너무 밝은 값을 부드럽게 눌러 흰색 번짐 방지
비유 — 사진 찍고 나서 인스타 필터를 입히는 단계예요. 형태는 그대로, 분위기를 확 끌어올립니다.

실제로 우리가 "왜 이렇게 하얗게 나오지?" 문제를 겪은 게 바로 이 톤보정이 빠져서였습니다.

// 발광 + 톤보정까지 = 최종 화면
밝은 곳만 번지게(블룸) → 톤 압축
밝은부분 = 색에서 일정 밝기 이상만;
번짐 = 흐리게(밝은부분);          // 발광
최종 = 톤매핑(색 + 번짐);          // 흰색 폭발 방지
정리

12겹이 모여 하나의 살아있는 풍경

마법은 없었습니다. 전부 "숫자를 매 프레임 GPU가 계산"한 결과예요.

  • 텍스처는 그림이 아니라 숫자표 (높이·색·흐름)
  • 셰이더가 픽셀·점마다 병렬로 계산
  • 각 효과는 얇은 한 겹, 합쳐져서 깊이가 됨
  • 변하는 모든 값은 천천히 섞여서(보간) 자연스러움

왼쪽 위 BUILD STACK 패널에서 RESET → NEXT를 누르면 이 12겹을 실제로 한 겹씩 켜볼 수 있습니다.
